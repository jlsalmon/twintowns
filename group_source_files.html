    • index.php
      Main application file. Instantiates tabbed interface, which loads site content asynchronously. Purely written in HTML5 and CSS3.

      http://www.cems.uwe.ac.uk/~jl2-salmon/dsa/index.phps

    • lib.php
      Contains functions for loading/validating config file and retrieving remote files from behind UWE proxy.

      http://www.cems.uwe.ac.uk/~jl2-salmon/dsa/lib.phps

    • style.css
      Global CSS styles for all content.

      http://www.cems.uwe.ac.uk/~jl2-salmon/dsa/css/style.css

    • config.xsd
      XML Schema for the configuration file. The config file must validate against the config file or the site will not load.

      http://www.cems.uwe.ac.uk/~jl2-salmon/dsa/config.xsd

    • town-chooser.php
      Allows hot-swapping between towns and twins. Drop-down boxes are displayed, which contain lists of towns and their respective twins. Selecting a new town/twin will cause the page data to be switched to the new selection via AJAX. This file populates the town/twin chooser drop-down boxes based upon the current session.

      http://www.cems.uwe.ac.uk/~jl2-salmon/dsa/town-chooser.phps

    • town-chooser.js
      Contains methods for updating PHP session variables via AJAX. These methods are called when the values of the town/twin selection drop-down boxes at the top of the page are changed.

      http://www.cems.uwe.ac.uk/~jl2-salmon/dsa/js/town-chooser.js

    • process.php
      This file is called via AJAX when a new town/twin is selected, and sets variables in the PHP $_SESSION array so that the page can be refreshed and content loaded based upon the new town/twin.

      http://www.cems.uwe.ac.uk/~jl2-salmon/dsa/process.phps

  • Component: Maps

    Contributors:
    • 10010381
    Files:
    • maps.php
      Creates the basic HTML structure for the maps page, inserts the google maps script, gets the XML config using ajax and calls the initialize() function in maps.js to load the maps.

      http://www.cems.uwe.ac.uk/~jl2-salmon/dsa/maps.phps

    • maps.js
      Loads the maps object, along with the markers and controls all map functionality. Map functionality includes:
      • initialize the map and its markers
      • toggle the display of markers on the map
      • toggle the display marker labels (on hover) on the map
      • initialize the markers (used for map initialization and when you switch town)
      • get info for a point of interest (when you click on a marker)
      • switch to a new town
      • swtich to a new twin

      http://www.cems.uwe.ac.uk/~jl2-salmon/dsa/js/maps.js

    • maps.css
      CSS stylesheet for the maps page

      http://www.cems.uwe.ac.uk/~jl2-salmon/dsa/css/maps.css

    • queryLP.phps
      Retrieves point of interest data from the Lonely Planet API.
      • sets up a HTTP request including Basic Authentication in the header
      • called from javascript and returns response

      http://www.cems.uwe.ac.uk/~jl2-salmon/dsa/queryLP.phps

  • Component: Weather

    Contributors:
    • 10000937
    Files:
    • weather.php
      Uses XSLT to transform a remote XML feed into HTML. Displays current weather conditions, astronomical data, current satellite image of the area, and a 10-day forecast for both town and twin. Note: This file should not be executed directly, but accessed via the Weather tab in index.php.

      http://www.cems.uwe.ac.uk/~jl2-salmon/dsa/weather.phps

    • weather_box.php
      Displays a small weather box containing current conditions and a 4-day forecast. Note: This file should not be executed directly, but accessed via the Home tab in index.php.

      http://www.cems.uwe.ac.uk/~jl2-salmon/dsa/weather_box.phps

    • wlive_town.xsl
